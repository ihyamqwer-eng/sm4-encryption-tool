# SM4 Encryption Tool — 国标SM4文件加解密工具

基于 GB/T 32907-2016 SM4 国密算法的文件加密/解密桌面工具。支持五种分组加密模式，提供桌面 GUI 和 Web 后端双操作路径。

## 核心原理

### SM4 国密算法（GB/T 32907-2016）

本工具实现了完整的 SM4 分组密码算法：

- **分组大小**：128 位（16 字节）
- **密钥长度**：128 位（16 字节）
- **轮数**：32 轮
- **结构**：Feistel 网络结构

算法由 C 语言实现（`c_sm4/sm4.c`），包含：

| 组件 | 说明 |
|------|------|
| **S-Box** | 256 字节非线性替换表，SM4 安全性的核心 |
| **FK/CK 参数** | 系统参数和固定参数，用于密钥扩展 |
| **T 变换** | S-Box + 线性变换 L（32 位循环左移 2/10/18/24 位异或） |
| **密钥扩展** | 32 轮迭代生成轮密钥 rk[32] |

### 五种分组加密模式

| 模式 | 特点 | 用途 |
|------|------|------|
| **ECB** | 每块独立加密，并行处理 | 算法教学，不推荐实际使用 |
| **CBC** | 链式异或前一块密文 | 文件加密标准模式，本工具默认 |
| **CFB** | 可加密少于 16 字节的数据 | 流式数据加密 |
| **OFB** | 加密过程不依赖明文 | 高噪声信道 |
| **CTR** | 可并行，计数器驱动 | 高性能场景 |

### Python 绑定

Python 通过 `ctypes` 调用动态链接库（`sm4.dll`）中的 C 函数：

```
Python 层:  encrypt_file(key, input_path, output_path)
             → ctypes 参数编排
                ↓
C 层:       sm4_encrypt_file(key_ptr, in_file_cstr, out_file_cstr, mode)
             → 文件 I/O + PKCS7 填充 + 分组加密
                ↓
SM4 核心:   sm4_setkey_enc() → sm4_crypt_cbc() → sm4_crypt_ecb()
```

文件加密时按块读入，自动 PKCS7 填充（16 字节对齐），并在密文头部写入 IV + 原始长度用于解密恢复。

### 加密文件格式

```
┌─────────────┐
│ IV (16B)    │ ← 随机初始化向量
├─────────────┤
│ 原始大小    │ ← size_t 文件长度
├─────────────┤
│ 密文数据    │ ← sm4_crypt_cbc 加密结果
└─────────────┘
```

## 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 加密算法 | **C 语言** | SM4 国标完整实现 |
| 绑定层 | **ctypes** | Python 直接调用 C DLL，零额外依赖 |
| 桌面 GUI | **Tkinter** | 自定义导航栏、现代按钮，暗红主题 |
| Web 后端 | **Flask + CORS** | 文件上传/解密 REST API |
| 密钥管理 | **SQLite** | 密钥持久化存储，CRUD + 模式迁移 |
| 文件管理 | **Python os** | 加密/解密/删除/清空 文件操作 |
| 构建 | **CMake / GCC** | 跨平台 C 编译，Windows DLL 导出 |

## 快速开始

```bash
# 1. 安装 Python 依赖
pip install flask flask-cors

# 2. 编译 C 代码（或使用预构建 sm4.dll）
cd c_sm4
gcc -shared -o sm4.dll sm4.c -DSM4_EXPORTS
cd ..

# 3. 启动
python app.py
```

启动后桌面 GUI 自动打开，Web 界面访问 http://localhost:5000
